# Log di Sviluppo - Versione 0.4.0

## Refactoring Strutturale: Verso un'Architettura Modulare

Questa versione non introduce nuove funzionalità di gioco, ma segna un'evoluzione tecnica fondamentale per la salute e la longevità del progetto. L'intera base di codice che gestisce i dati di gioco è stata sottoposta a un massiccio refactoring per passare da un'architettura monolitica a una modulare.

### Motivazione

Con l'espansione del gioco, il file `game/gameData.ts` era diventato un monolite contenente la logica di tutte le stanze, rendendolo difficile da leggere, manutenere e scalare. Qualsiasi modifica richiedeva di navigare un file di centinaia di righe, aumentando il rischio di errori e regressioni.

### Modifiche Chiave

1.  **Modularizzazione per Stanza**:
    -   È stata creata una nuova directory `game/rooms/`.
    -   La logica di ogni singola stanza del gioco è stata estratta dal file monolitico e spostata in un proprio file dedicato all'interno di questa directory (es. `plancia.ts`, `stiva.ts`, `corridoioPrincipale.ts`, etc.).

2.  **Aggregatore di Moduli**:
    -   È stato creato un file `game/rooms/index.ts` che funge da aggregatore. Questo file importa tutte le stanze dai loro rispettivi moduli e le esporta in un unico oggetto `allRooms`, mantenendo la stessa struttura dati che il motore di gioco si aspetta.

3.  **Centralizzazione dei Tipi**:
    -   Le definizioni di tipo comuni (`Room`, `Command`, `PlayerState`, etc.) sono state spostate nel file `types.ts` per essere importate in modo coerente da tutti i nuovi moduli, evitando duplicazioni e migliorando l'organizzazione.

### Vantaggi Ottenuti

-   **Migliore Manutenibilità**: Modificare o correggere il bug di una stanza ora richiede di intervenire solo su un piccolo file isolato.
-   **Scalabilità Semplificata**: Aggiungere nuove stanze al gioco è ora un processo molto più pulito: basta creare un nuovo file e aggiungerlo all'aggregatore.
-   **Leggibilità del Codice**: File più piccoli e focalizzati su una singola responsabilità sono intrinsecamente più facili da capire e da navigare.
-   **Nessuna Regressione**: Il refactoring è stato puramente strutturale. Il comportamento del gioco, la narrazione e le meccaniche sono rimasti assolutamente identici, garantendo che l'esperienza del giocatore non sia stata alterata.

Con questa modifica, la base di codice de "IL RELITTO SILENTE" è ora molto più robusta, professionale e pronta per eventuali espansioni future.
